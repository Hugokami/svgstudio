## $(date +%Y-%m-%d) - Optimize Marquee Selection Array Inclusion Check

**Learning:** When a marquee selection logic compares a new set of elements (`newSelections`) against an existing tracked array (`selectedElements`) inside a loop using `Array.includes()`, it creates an $O(N \times M)$ algorithmic bottleneck. This is common in drag-to-select DOM interactions where large numbers of elements might overlap the selection box during high-frequency mouse moves.
**Action:** When performing membership checks inside a loop against an existing array that is also being modified during the iteration, instantiate a `Set` from the array before the loop. Perform $O(1)$ membership checks using `Set.has()` and ensure both the `Set` and the original array are updated synchronously inside the loop. This reduces complexity to $O(N + M)$ while preserving array order and references.
