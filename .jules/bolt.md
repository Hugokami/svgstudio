## 2024-05-18 - SVG CloneNode Optimization
**Learning:** In large SVG manipulation files like `js/app.js` and `svgviewer.html`, repeatedly creating SVG elements (e.g., control handles) in a loop using `document.createElementNS` is significantly slower than configuring a single static template outside the loop and cloning it with `.cloneNode(true)`. However, doing this for a small number of elements can be considered a micro-optimization. Avoid polluting the global `window` scope when creating template references.
**Action:** When adding multiple repeating visual elements (like handles or nodes) to the DOM, create a static template variable in the module closure, pre-configure it with static attributes, and use `.cloneNode(true)` inside the loop, only updating dynamic data (like coordinates).
